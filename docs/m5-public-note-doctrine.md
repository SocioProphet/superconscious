# M5 Public Note Doctrine

## Status

M5 turns the certificate outcome into a Colah-shaped public note.

The public note is not written after deciding which story is rhetorically convenient. The certified outcome selects the template.

## Single conceptual move

```text
Feature steering needs an implementability test.
```

For off-target failure outcomes, the move sharpens to:

```text
Feature steering needs an off-target audit before it can be treated as safety-preserving.
```

## Template discipline

Five templates are pre-drafted:

```text
templates/m5/public-note-pattern-a-clean.md
templates/m5/public-note-pattern-b-clean.md
templates/m5/public-note-pattern-c-clean.md
templates/m5/public-note-off-target-damage.md
templates/m5/public-note-no-viable-feature.md
```

The certificate outcome selects the template. Runtime authors do not get to silently substitute a more favorable frame.

## Figure discipline

The note must contain one load-bearing figure. For successful or boundary/out-of-distribution outcomes, this is the five-curve implementability plot or the M3 cross-layer extension. For off-target-damage outcomes, the load-bearing figure is the contrast between on-target gain and off-target safety regression. For no-feature outcomes, it is the feature-funnel attrition plot.

## Synthetic boundary

The synthetic M5 fixture validates schema and composition only. It is not a public empirical note.

## Publication requirements

A runtime public note must include:

- selected template;
- M1/M2/M3 certificate references;
- load-bearing figure path;
- five-curve or cross-layer figure path;
- limitations;
- non-claims;
- outcome-specific interpretation.
