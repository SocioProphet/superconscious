# Neuro-Symbolic Cognition Position

Status: v0.1 doctrine capture.

This document defines how Superconscious consumes neuro-symbolic reasoning capabilities after the CHRONOS / ASU neuro-symbolic alignment tranche landed in SocioSphere.

It does not add runtime authority. It does not authorize model calls, network calls, external tools, shell execution, browser automation, durable memory promotion, schema promotion, policy admission, or workspace mutation.

## Position

Superconscious is the visible governed cognition loop over neuro-symbolic capabilities.

Superconscious coordinates and records. It does not authorize.

It may coordinate, inspect, classify, request, record, and emit safe operational traces. It must not become the symbolic reasoner, ontology authority, schema authority, policy authority, model-routing authority, memory authority, execution authority, or evidence/replay authority.

The governing sentence is:

```text
Superconscious coordinates neuro-symbolic cognition; it does not authorize neuro-symbolic claims.
```

## Relationship to CHRONOS

CHRONOS is the governed evidence lifecycle around neuro-symbolic work:

```text
source corpus / observation
  -> evidence anchor
  -> normalized claim
  -> candidate governed carrier
  -> explanation trace
  -> verification result
  -> governance decision
  -> receipt / rejection
  -> learning update
```

Superconscious may invoke or record this lifecycle in a safe operational trace, but it does not own the carrier's final authority.

## Superconscious responsibilities

Superconscious may:

- identify that a task requires neuro-symbolic handling;
- classify the method family being requested or observed;
- request policy admission before any tool, model, memory, route, or runtime action;
- request evidence anchoring and replay through AgentPlane-compatible surfaces;
- record method outputs as candidate artifacts;
- emit risk signals when authority drift is detected;
- request policy review for ambiguous or elevated-risk claims;
- record learning-loop proposals without promoting memory.

## Superconscious prohibitions

Superconscious must not:

- treat model output as evidence;
- treat fuzzy scores as truth;
- treat stable-model output as execution permission;
- treat learned rules as canonical schemas;
- treat ontology embeddings as ontology authority;
- treat symbolic policies as live controllers;
- persist symbolic memory without governance;
- call a model provider directly;
- bypass model router, policy fabric, AgentPlane, Ontogenesis, sourceos-spec, or model-governance-ledger.

The carriers Superconscious observes — a fuzzy score, a neural output, a learned rule, a symbolic derivation — are advisory references only. A neural output is not evidence and a symbolic derivation is not policy admission; final authority stays with AgentPlane, Ontogenesis, and sourceos-spec.

## Safe trace requirements

When Superconscious records a neuro-symbolic event, the safe trace should preserve:

- method family;
- source evidence reference or explicit missing-evidence risk;
- carrier status;
- grounding status;
- validation status;
- policy decision reference or pending state;
- owning authority plane;
- non-authority declaration;
- replay reference when available.

The safe trace must not expose raw private chain-of-thought. It should record public-safe summaries, decision references, provenance references, and artifact hashes.
