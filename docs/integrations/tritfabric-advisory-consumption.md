# TritFabric advisory consumption boundary

## Status

Contract and boundary registration only.

Superconscious may consume TritFabric-derived Community Learning, mentor/learner, model-card, and readiness semantics only as coordinator/advisory context. It is not the authority plane for TritFabric contracts, Community Learning policy, Ontogenesis vocabulary, Sociosphere estate registration, model promotion, or Serve deployment.

## Upstream authority map

- TritFabric owns implementation contracts and immediate model/community/serve surfaces.
- Ontogenesis owns RDF/OWL/JSON-LD/SHACL vocabulary and gates.
- Sociosphere owns estate registration and cross-repo routing.
- Prophet Platform owns product/runtime consumption planning.
- Superconscious owns advisory coordination, reflection, and reasoning-loop support only.

## Allowed use

Superconscious may use TritFabric-derived semantics to:

- label mentor/learner/coordinator reasoning context;
- surface advisory warnings about missing consent, license, lineage, rubric, or manual review;
- refer to Network Atlas catalog status as advisory context;
- represent readiness/reporting surfaces as non-authoritative evidence;
- route users toward the owning authority plane.

## Prohibited authority drift

Superconscious must not:

- approve or execute model promotion;
- validate framework adapters;
- adjudicate Community Learning eligibility as the final authority;
- create transferable or economic credit;
- deploy or enable Serve autoscaling;
- override Ontogenesis SHACL vocabulary or TritFabric contract gates;
- treat advisory coordinator state as an execution authorization.

## Claim boundary

This document does not implement runtime integrations, model training, workflow execution, artifact promotion, adapter validation, or Serve deployment. It records the advisory consumption boundary only.
