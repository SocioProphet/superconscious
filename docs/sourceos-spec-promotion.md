# SourceOS Spec Promotion Plan

Superconscious may stage local draft artifact shapes only to harden the loop. Canonical contract ownership belongs in `SourceOS-Linux/sourceos-spec`.

## Current promotion state

The first SourceOS reasoning contract tranche has been promoted upstream:

| Canonical schema | Purpose | Compatibility artifact |
|---|---|---|
| `ReasoningRun` | Top-level governed cognition run record. | `reasoning-run.json` → `reasoning-run.sourceos.json` |
| `ReasoningEvent` | Safe operational trace event. | `events.jsonl` → `reasoning-events.sourceos.jsonl` |
| `ReasoningReceipt` | Final run receipt with trace hash, coordination posture, and replay class. | `agentplane-evidence.json` → `reasoning-receipt.json` |
| `ReasoningReplayPlan` | Replay class, inputs, and constraints. | `replay-plan.json` → `reasoning-replay-plan.json` |
| `ReasoningBenchmark` | Evidence-backed benchmark result. | `benchmark-result.json` → `reasoning-benchmark.json` |

The Superconscious canonicalization layer now emits both compatibility artifacts and SourceOS-shaped artifacts. This lets AgentPlane, SocioSphere, sourceosctl, Agent Machine, product surfaces, and benchmark tooling move toward canonical contract names without breaking the M1 loop.

## Local compatibility artifacts

M1 continues to emit these compatibility files:

```text
events.jsonl
reasoning-run.json
agentplane-evidence.json
replay-plan.json
benchmark-result.json
```

## Canonical SourceOS artifacts

After canonicalization, each run directory also contains:

```text
reasoning-events.sourceos.jsonl
reasoning-run.sourceos.json
reasoning-receipt.json
reasoning-replay-plan.json
reasoning-benchmark.json
```

Run:

```bash
make canonicalize
```

or directly:

```bash
python3 packages/superconscious-core/superconscious_core/canonicalize_artifacts.py .runs/<run-id>
```

## Required upstream additions still remaining

The first tranche does not complete the whole contract surface. Remaining work:

- JSON-LD context entries for the new reasoning schemas.
- Hydra resource entries if reasoning artifacts become API-addressable.
- OpenAPI patch entries if the platform exposes reasoning run endpoints.
- AsyncAPI channel fragments for streamed reasoning events.
- Catalog README updates in `sourceos-spec` if not already generated by automation.
- A narrower canonical adapter record object, if `ReasoningRun.adapterRecords` proves too loose.

## Deferred object

The local `AdapterDecision` draft object remains useful inside Superconscious, but direct promotion was blocked by connector filtering during the first tranche. The upstream schema currently models adapter records as structured objects in `ReasoningRun.adapterRecords` and coordination summaries in `ReasoningReceipt.coordination`.

Future candidate name:

```text
ReasoningStepRecord
```

This should be considered only after AgentPlane, SocioSphere, Guardrail Fabric, Agent Registry, and Model Router consume the first tranche.

## Compatibility rules

- All canonical IDs use stable `urn:srcos:<type>:<slug-or-hash>` forms.
- Superconscious local draft schemas remain validation aids, not canonical authority.
- SourceOS canonical artifacts should not require raw private reasoning content.
- Product surfaces should render safe operational traces only.
- Consumers should accept compatibility artifacts during M1 and prefer canonical artifacts for new integrations.

## Next promotion batch
